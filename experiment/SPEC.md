# 実装課題: 類似記事クラスタリング

このリポジトリ（IndiviNEWS）に、複数ニュースソース間で「同じ話題」の記事を
まとめるクラスタリング機能を追加してください。

## 背景

Hacker News / はてなブックマーク / Zenn / Publickey から見出しを集めているが、
同じ記事・同じ話題が複数ソースに重複して現れる（例: Zennの記事がはてなブックマークにも
流れてくる、同じ発表がHacker Newsとはてなの両方で話題になる）。現在はURL完全一致の
重複排除しかなく、URLが少しでも違う同一記事や、URLが違う同一話題はまとめられていない。

## 成果物1: クラスタリングモジュール

`src/cluster.js` を新規作成し、次の関数をエクスポートすること。

```js
/**
 * @param {Array<{source: string, title: string, url: string, publishedAt: Date|string}>} items
 * @returns {Array<Array<item>>}  クラスタの配列。各クラスタは渡されたitemオブジェクトの配列。
 */
function clusterItems(items) { ... }
module.exports = { clusterItems };
```

要件:

- 「同じ記事」または「同じ話題」と判断できるアイテム同士を同一クラスタにまとめる。
  単独の記事は要素1のクラスタとして返す。全アイテムがちょうど1つのクラスタに属すること。
- 返す配列内のアイテムは、**渡されたオブジェクトそのもの**を使うこと（コピー不可）。
- 各クラスタ内の並びは新しい順、クラスタ同士の並びは代表記事（クラスタ内で最新のもの）の新しい順。
- 決定的であること（同じ入力には常に同じ出力）。
- 少なくとも次のようなケースを正しく扱えること:
  - URLの表記ゆれ（クエリパラメータの有無、末尾スラッシュ、http/https など）で同一記事
  - 同一記事が別ソースに（タイトル末尾に「 - Qiita」などサイト名サフィックスが付く場合を含む）
  - タイトルの表記ゆれ（全角/半角、大文字/小文字、【速報】等の装飾）で同一話題
  - タイトルが部分的に似ているだけの**別の話題**を誤ってまとめないこと
    （例:「GPT-6を発表」と「GPT-6の安全性に懸念」は別話題）
- 対象言語は日本語と英語。ただし**言語をまたいだ**同一話題の検出（英語見出しと
  日本語見出しのマッチング）はスコープ外とする。

## 成果物2: ダイジェストへの組み込み

`scripts/build-digest.js` を修正し、`clusterItems` を使って:

- 複数ソースにまたがるクラスタ（要素2以上）があれば、代表記事1件のみを掲載し、
  末尾に `（HN・はてなで話題）` のように**まとまったソース名の一覧**を注記する。
- 既存のURL完全一致による重複排除ロジックはクラスタリングに置き換えてよい。
- 既存の出力構成（今日のピック / ソース別セクション）は維持すること。

## 制約

- **新しいnpm依存を追加しないこと**（Node.js標準ライブラリと既存依存のみ）。
- **LLM APIや外部ネットワーク呼び出しをモジュール内で行わないこと**（コスト0円の維持）。
- 想定入力は最大100件程度。パフォーマンスチューニングは不要。
- `experiment/` ディレクトリ配下のファイルは**読まない・変更しないこと**
  （このSPEC.md自体を除く）。採点に使うため、内容を見ると評価が無効になる。

## 完了条件

- `node scripts/build-digest.js` がエラーなく動作し、ダイジェストが生成される。
- ウィジェット表示（`api/index.js` / `dev-server.js`）が壊れていない。
- 実装の正しさを自分で検証すること。検証方法は任せる。
