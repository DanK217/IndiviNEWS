# 自走ニュースダイジェスト（LLM不使用・運用コスト0円）

GitHub Actionsが毎朝06:00 JSTに Hacker News / はてなブックマーク人気エントリー /
Zenn・Publickey から見出しを取得し、`digests/YYYY-MM-DD.md` としてリポジトリに
自動コミットする仕組み。

## 設計方針: なぜLLMを使わないのか

本リポジトリの要件は「ランニングコスト0円」。Claude等のLLM APIは
サブスクリプションプランとは別会計の従量課金であり、毎朝の自動実行に
組み込むと恒常的なコストが発生してしまう。

そこで「実行時」にはLLMを一切使わず、選定・整形はルールベースで行う:

- **今日のピック**: ウィジェット表示と同じ「ソース間交互選出」で上位3件
  （更新頻度の高いHacker Newsが他ソースを埋めないための工夫）
- **ソース別セクション**: 各ソースの最新5件
- **重複排除**: はてなブックマークにはZenn等の記事も流れてくるため、
  ソースをまたいだ同一URLは除外

LLM（Claude Code / Fable 5）が活躍したのは**構築時**。この仕組み自体の設計・
実装・検証をAIに任せ、成果物はAI無しで永続的に動く——という役割分担にしている。

## 運用コスト内訳

| 項目 | プラン | コスト |
|---|---|---|
| ウィジェット配信 | Vercel Hobby | 0円 |
| ダイジェスト生成 | GitHub Actions（プライベートリポジトリ無料枠 2,000分/月のうち毎日約1分） | 0円 |
| ニュース取得 | 各公開API/RSS（認証不要） | 0円 |
| 要約・選定 | ルールベース（LLM不使用） | 0円 |

シークレット登録（APIキー等）も一切不要。

## 使い方

```bash
# ローカルで手動生成
node scripts/build-digest.js
```

GitHub Actionsでは毎朝自動実行される。手動実行したい場合は
Actionsタブ → news-digest → Run workflow。

## 構成ファイル

- `scripts/build-digest.js` — ダイジェスト生成本体
- `src/fetchers/` — ニュース取得層（ウィジェットと共用）
- `.github/workflows/news-digest.yml` — 毎朝の自動実行とコミット
- `digests/` — 生成されたダイジェストの蓄積先
